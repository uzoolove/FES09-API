  • [회원] 판매자로 승급 요청
    • API 문서 참고
    • 이전의 회원 승인 방법은 extra.confirm 값을 false로 지정했을 때 비승인 상태로 로그인을 못하게 제약할 수 있는데 관리자가 true로 변경하면 승인 처리가 되면서 로그인이 가능
    • 이를 판매 회원에만(type=seller) 적용해서 로그인을 못하게 하고 일반 회원에는(type=user) extra.confirm 값이 false일 경우에도 로그인이 가능하게 수정함
    • 일반 회원이 판매 회원으로 승급(?)하기 위해서 클라이언트 측에서 구현할 작업으로, 승급 조건이 맞으면 승급 요청시 extra.confirm 값을 false로 수정하고 이를 승인하기 위해서 관리자는 extra.confirm이 false 회원을 검색해서 이 값을 true로 변경하고 type을 "seller"로 수정하도록 개발해야 함
